iOS Base Project Readme

The base project is a great way to start new iOS projects. It includes some of the best and most popular cocoapod libraries available. We have also included some sample codes for good design patterns and documentation.

How to use?

You can rename the project, or simply use the podfile when starting your own new projects.

Design patterns

Software design patterns enable more organized and usable code. Expand beyond Model View Controller (MVC) as needed.

POP (Protocol Oriented Programming)

Flexible pattern that produce more re-usable code than Objeft Oriented Programming.

  • Types can conform to multiple protocols.
  • Can be adopted by Class, Struct, and Enum.
  • Can include function implementation in protocol extensions.

MMVM (Model View View Model)

  • The View Model should hold logic and data used by the views. This separation of concern helps to reduce massive view controllers.
  • Easier to write unit tests on view model logic. #### Reactive
  • Automatically update views as data model changes state. This technique makes the UI more responsive.
  • The controller can use 2 way binding via didSet to reduce logic. #### Protocol Oriented Programming
  • Think about protocol oriented code with value semantics for models and view models.
